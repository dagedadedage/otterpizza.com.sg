# 2026-06-25 Devlog — Otter Pizza

## Google Safe Browsing Fix
- Root cause: `robots.txt` blocked all crawlers (`Disallow: /`) + `noindex, nofollow` on every page + Google OAuth redirect looked like phishing
- Fixed: robots.txt allows crawling (blocks only `/admin/` `/api/` `/checkout/`), meta robots set to `index, follow`, privacy page at `/privacy`, security.txt at `/.well-known/security.txt`, sitemap.xml
- Added Nginx security headers: X-Content-Type-Options, X-Frame-Options, X-XSS-Protection, Referrer-Policy, Permissions-Policy
- Restaurant JSON-LD schema on all pages with OG/Twitter card metadata

## Session Persistence (Major Fix)
- Switched from custom JWT encode/decode to NextAuth native JWE-encrypted tokens
- Removed all custom `signToken`/`verifyToken` bridge — NextAuth handles entire lifecycle
- Session expiry: 15 days, `refetchOnWindowFocus: false`, `refetchInterval: 0`
- Login page auto-redirects to admin if already authenticated
- Logout: server-side `GET /api/admin/auth/logout` clears all cookies (no CSRF needed)
- `LegacyCookieCleanup` removes old `otter-admin-token` cookies

## SEO Migration Prep
- Scanned Wix site (otterpizza.com.sg): 31 products, 9 stores, URL structure documented
- Complete redirect map: all Wix `/product-page/{slug}` → Next.js `/menu/[slug]`
- All 8 store coordinates verified via OneMap API geocoding
- Full reference saved to `memory/wix-seo-migration.md`

## Order Flow Simplification
- Removed ACCEPTED status — PAID flows directly to READY or OUT_FOR_DELIVERY

- Context-aware status buttons: Pickup orders show "Ready for Pick-up", Delivery orders show "Out for Delivery"
- API enforces delivery-type-aware transitions (blocks READY for delivery, blocks OUT_FOR_DELIVERY for pickup)
- Auto-transition: saving tracking URL on READY delivery order → OUT_FOR_DELIVERY
