## Webhook — Major Debugging Session
- HitPay sends TWO webhook types to same endpoint:
  1. Payment-request callback: HMAC in JSON body `hmac` field, signed with API key
  2. Event webhook (dashboard): signature in `hitpay-signature` header, per-webhook secret
- Old handler only checked headers → missed body `hmac` entirely
- Fix: checks header sig first, then body `hmac`, tries both API key + webhook salt
- HitPay API fallback: when HMAC fails, calls `getPaymentStatus` to verify independently
- 5 stuck orders manually recovered (webhook HMAC mismatch during debugging)
